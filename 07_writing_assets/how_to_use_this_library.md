# 如何使用这个写作资产库

> 第一次使用建议先看：`START_HERE_先看这个.md`。如果你已经知道当前任务，直接看：`任务导航_我现在该看哪个文件.md`。

## 1. 有自己的实验结果时
1. 写下 3 个核心 claim，每个 claim 必须能对应一个图或表。
2. 在 `related_work_matrix.csv` 中按 topic、contribution_type、evidence_type 过滤出 8-12 篇最相近论文。
3. 用这些论文建立自己的 related work 表：prior work、方法、平台、指标、局限、你的区别。
4. 参考 `venue_pattern_matrix.csv` 选择目标 venue。
5. 从 `paper_skeleton_templates.md` 选一个骨架。
6. 用 `abstract_intro_sentence_bank.md` 写摘要和 Introduction 初稿。
7. 用 `reviewer_checklist.md` 做投稿前自查。

## 2. 只有方向、还没有结果时
1. 先在矩阵里找 topic 相近论文，观察它们的 limitation。
2. 把高频 limitation 转成可实验验证的 gap。
3. 设计一个最小实验，让结果能填进 SOTA 表。

## 3. 只想练某一种写法时
1. 每次只选 1-2 篇与你当前任务最相关的论文。
2. 只看 abstract、Introduction、main architecture figure、SOTA table。
3. 用句式库重写一版摘要，不看原文复述。
4. 把图表套路记录到自己的论文草稿模板。

## 4. 以后扩展专题库
这 60 篇是写法层。真正写某个方向时，再单独建 30-50 篇方向专题库，例如 ROTRNG、SM4 crypto accelerator、CIM、RISC-V SoC 或 FPGA DNN accelerator。
