# 常见看不懂点解释

## gap

gap 是“现有论文还没解决、而你能用实验验证的问题”。

不是 gap：
- 这个方向很重要。
- 现有方法还有不足。

更像 gap：
- 现有 FPGA NTT 加速器提升吞吐，但在 resource-normalized efficiency 上对比不足。
- 现有 PUF 防护方法提高安全性，但 CRP discard 带来可用性损失。

## claim

claim 是你要让审稿人相信的结论。

claim 必须有证据。没有图表支撑的句子不要放进摘要。

## baseline

baseline 是你拿来比较的对象。

好的 baseline：
- 同任务。
- 同平台或可归一化。
- 最近几年。
- 指标完整。

## SOTA

SOTA 是 state of the art，表示当前同类工作中比较强的方案。

注意：
SOTA 表不是越多越好，而是要公平。FPGA、ASIC、CPU、GPU 不能不解释就混在一起比较。

## ablation

ablation 是消融实验，用来证明你的每个模块都有用。

例子：
- baseline。
- baseline + pipeline。
- baseline + memory banking。
- full design。

## contribution

contribution 不是“我做了什么”，而是“我做的东西解决了什么问题，并被什么证据证明”。

## limitation

limitation 是适用边界。

写 limitation 不是示弱，而是告诉审稿人你知道自己的 claim 在哪里成立。

## evaluation

evaluation 是实验验证。

数字 IC / FPGA 论文常见 evaluation：
- resource / area。
- frequency。
- latency。
- throughput。
- power / energy。
- accuracy。
- security metric。
- runtime / QoR。

## normalized metric

归一化指标用于公平比较。

例子：
- throughput per LUT。
- energy per operation。
- area-time product。
- throughput per DSP。

## artifact

artifact 是能支撑论文结果的工程产物，例如 RTL、脚本、测试向量、benchmark、数据集、版图或测量数据。
