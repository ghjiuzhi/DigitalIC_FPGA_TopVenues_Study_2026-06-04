# 论文阅读卡片模板

这个模板的目的不是让你全文翻译论文，而是让你每读一篇都能留下“写论文时能用”的东西。

## 使用方法

每篇论文只填下面 8 个问题。填不出来的地方先空着，不要卡住。

## 阅读卡片

### 1. 这篇论文一句话解决什么问题？

模板：
这篇论文针对 `[对象/场景]` 中的 `[瓶颈]`，提出 `[方法/架构/电路/工具]` 来改善 `[指标]`。

### 2. 它为什么属于数字 IC / FPGA 方向？

看它是否涉及：
- FPGA / ASIC / CMOS / SoC
- architecture / circuit / EDA
- resource / area / power / latency / throughput
- security hardware / cryptographic hardware

### 3. 摘要是怎么排顺序的？

记录顺序：
问题 -> 方法 -> 平台 -> 指标 -> 对比 -> 结论

### 4. Introduction 的 gap 是什么？

不要写“现有方法不好”。要写具体：
- 哪个指标不够？
- 哪个场景没覆盖？
- 哪个假设不现实？
- 哪个平台没验证？

### 5. 方法图讲了什么？

看图时只问：
- 输入是什么？
- 输出是什么？
- 哪个模块是本文新增？
- 数据怎么流？
- 瓶颈在哪里被解决？

### 6. 实验表证明了什么？

记录：
- baseline 是谁？
- 平台是什么？
- 指标是什么？
- 提升幅度是什么？
- 有没有归一化？

### 7. 这篇最值得我模仿什么？

从里面选 1-2 个：
- 摘要句式
- gap 写法
- 方法图
- 实验表
- SOTA 对比
- limitation 写法

### 8. 这篇有什么局限？

常见局限：
- 只在一个 FPGA 上验证。
- 没有功耗。
- 没有 ASIC 后端。
- baseline 少。
- 只做 simulation。
- 安全假设较强。

## 填完后怎么用

把第 4 点变成 related work 的 gap，把第 7 点放进自己的写作素材库，把第 8 点变成自己论文要避开的坑。
