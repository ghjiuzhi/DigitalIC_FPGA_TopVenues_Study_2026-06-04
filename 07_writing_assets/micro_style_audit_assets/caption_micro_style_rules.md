# Caption Micro Style Rules

## 好 caption 的四个要素

1. What：图表展示什么。
2. Setup：数据来自什么平台、样本或条件。
3. Claim：它支撑哪个结论。
4. Boundary：它不支撑什么更大的说法。

## 可用模板

`[Figure/Table] shows [observable] under [setup]. The result supports [claim] by [reason]. It should be interpreted within [boundary].`

## 数字表 caption

数字表 caption 不要只写表名。应说明：

- 是否 representative、complete、repeat summary 或 archive summary。
- 样本数量和条件。
- 数字如何计算。
- 是否用于主 claim 或控制实验。

## RO-TRNG 当前稿件规则

- TDC caption 要写 diagnostic，不写 calibrated physical model。
- NIST/SP800 caption 要写 assessment summary，不写 certification。
- TVLSI-track caption 要写 imported diagnostic evidence，不写 merged manuscript claim。
- Toolflow caption 要写 sensitivity matrix，不写 universal tool conclusion。
